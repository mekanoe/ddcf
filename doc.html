DOCUMENTATION

Format


Just so you know what to expect here, I'll give a basic and straightforward explaination and then a technical explaination. Generally speaking, the first one should tide you over.

SETUP

Setup is done around line 16 on the script. Be sure to set it before you run the program.
1: Your E-mail Address 
emailAddress = YOUR CLOUDFLARE EMAIL HERE
2: Your API Key 
APIkey = YOUR CLOUDFLARE API KEY HERE

These are the only two you need to worry about, the only other option you should change is...
Default Hostname 
defaultHostname = YOUR DEFAULT DOMAIN

If you wish to change the last option, read this.

COMMANDS

Update



Invokation: 
$ ddcf update
Full syntax: 
$ ddcf u|update [hostname]
This command will update the hostname passed as an argument or the default hostname to your current IP. This will only work on A records, however will return "Success." if you try to change a CNAME. Contrarily to that, it will not, however, update the record.

This command will get your IP from a web script (specifically and by default `http://canhazip.com') and makes the CloudFlare API call to update the DNS entry via HTTPS.

Force



Invokation: 
$ ddcf force 127.0.0.1
Full syntax: 
$ ddcf f|force value [hostname]
This command will update the hostname passed as an argument or the default hostname to the value argument.

This command does the same thing as `update', except it does not check for your current IP. This might work to change CNAME records, however it has not been tested to do so.

Stats



Invokation: 
$ ddcf stats
Full syntax: 
$ ddcf s|stats [interval]
This will show you some basic information about your account's statistics. Everything interesting that it can show is there.

This command takes all the interesting information from the stats API call, does some math and CLI formatting, and displays it all pretty. Due to a bug in CloudFlare's Client API, this does not currently support showing per-zone (hostname) stats. As soon as the API is updated, I will add this capability.

Add



Invokation: 
$ ddcf add example.com sub A 127.0.0.1
Full syntax: 
$ ddcf a|add domain|X name A|CNAME value|X [on|off]
This command will add name to domain as either an A or CNAME record with value of value. If value is X, it will get your current IP and use it. This only will work with A records. If domain is X, it will use the default hostname. Note, this will fail if your default hostname is not the root domain name. Value X does the same operation as `update', however adds a record rather than updating it.

If the last argument is passed, on means a happy orange cloud[sic] and off means a sad grey cloud.

Remove



Invokation: 
$ ddcf remove example.com sub.example.com
Full syntax: 
$ ddcf r|rm|remove domain|X record
This command will remove the record argument. This will remove all records for that record (including MX, TXT, SPF, etc). If hostname is set to `X', it will use the default hostname.
Record must be set to the full DNS entry.
Example: `sub.example.com'
NOT: `sub'

This command will remove the record argument via API call.
